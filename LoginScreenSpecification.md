# 画面仕様書

## 画面ID: SCR-01
## 画面名: ログイン画面

## 目的
未ログインのユーザに対し、Googleアカウントを用いたログイン(OAuth認証)を提供し、認証成功後にホーム画面へ遷移させる。

## レイアウト

```
+------------------------------------------------------------+
|                                                              |
|                                                              |
|                   サービス名 / キャッチコピー                 |
|                                                              |
|                                                              |
|                 +--------------------------+                |
|                 |   Googleでログイン        |                |
|                 +--------------------------+                |
|                                                              |
+------------------------------------------------------------+
```

中央:サービスロゴ、サービス名(またはキャッチコピー)、Googleログインボタンを縦に配置。

## ボタン・リンク

| ボタン名 | 動作 | 遷移先画面ID |
|---|---|---|
| Googleでログイン | Googleの認証画面(外部)へリダイレクトし、OAuth認証を実行する | 認証成功時: SCR-02(ホーム画面) |

## 表示項目

| 項目名 | データソース(情報モデル) |
|---|---|
| サービス名/キャッチコピー | 静的リソース |
| Googleでログインボタン | 静的リソース(OAuth連携設定) |

## エラー時挙動

- **認証キャンセル・拒否**: ユーザがGoogle認証画面で認証をキャンセルした場合、または権限付与を拒否した場合、本画面に留まり、エラーメッセージ(例:「ログインがキャンセルされました」)を表示する。
- **認証エラー**: Google側で認証処理に失敗した場合、エラーメッセージを表示し、再度「Googleでログイン」を押下できる状態に戻す。
- **通信エラー**: OAuth認証処理中にサーバとの通信に失敗した場合、エラーメッセージを表示し、再試行を促す。
- **新規ユーザ判定**: 認証成功時、ユーザ情報が未登録の場合は自動的にユーザ登録処理を行った上でSCR-02(ホーム画面)へ遷移する。登録済みの場合はそのままSCR-02へ遷移する。
